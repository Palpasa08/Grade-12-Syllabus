Database and DBMS Concepts

1. What is Database and DBMS?List out the advantages and disadvantages of DBMS.

Database: A database is an organized collection of data stored electronically, while a Database Management System (DBMS) is the software that manages and provides access to this data.

DBMS (Database Management System): A DBMS is a software system that allows users to interact with a database. It provides tools for creating, managing, and manipulating the data stored in the database. DBMS handles tasks like data storage, retrieval, security, and integrity.

The advantages of DBMS are:

  • Reduces data redundancy
  • Ensures data consistency and integrity
  • Allows data sharing among multiple users
  • Provides backup and recovery
  • Improves data security

The disadvantages of DBMS are:

  • High initial cost and complexity
  • Requires technical expertise
  • System failure can affect all users
  • Performance may be slower in large systems

2. Differentiate between file processing system and DBMS.

File Processing System DBMS
In a file processing system, data is stored in separate individual files. In a DBMS, data is stored in a centralized and organized database.
File processing systems often lead to high data redundancy due to duplicate data storage. DBMS minimizes data redundancy by maintaining a single, consistent data source.
It is difficult to enforce security in file processing systems because of isolated file management. DBMS provides better security features by managing user access and permissions centrally.
Retrieving and updating data is hard and time-consuming in file processing systems. DBMS allows easier and more efficient data retrieval and updating using structured queries.

3. Explain the different models of DBMS with advantages and disadvantages.

Hierarchical Model

The hierarchical model organizes data in a tree-like structure, where each record has a single parent and can have multiple children.

Advantages:

  • It provides simple and fast access to hierarchical and nested data.
  • Data integrity is maintained through parent-child relationships.

Disadvantages:

  • The model is inflexible and difficult to modify if the data structure changes.
  • It does not support many-to-many relationships naturally.

Network Model

The network model organizes data using a graph structure, allowing many-to-many relationships between records through sets.

Advantages:

  • It supports complex relationships more effectively than the hierarchical model.
  • Data access is fast due to multiple access paths.

Disadvantages:

  • The model is complex to design and manage.
  • It requires knowledge of data paths for querying.

Relational Model

The relational model stores data in tables consisting of rows and columns, and relationships are established through keys.

Advantages:

  • It is easy to use and understand due to its tabular format.
  • Flexible querying is possible using SQL.
  • Normalization reduces data redundancy.

Disadvantages:

  • It can be slower for handling highly complex or hierarchical data.
  • Requires more storage for maintaining relationships and indexes.

Object-Oriented Model

The object-oriented model represents data as objects, similar to those used in object-oriented programming languages.

Advantages:

  • It supports complex and multimedia data types effectively.
  • It allows better integration with object-oriented applications.

Disadvantages:

  • It is less widely adopted and supported compared to relational databases.
  • The design and implementation can be complex and resource-intensive.

4. What is a relational database? How is it different from other database models?

A Relational Database stores data in tables with rows and columns, establishing relationships between different pieces of information.

Differences from other models:

  • Uses tables, unlike hierarchical/tree or network/graph structures.
  • Allows flexible querying with SQL.
  • Supports normalization to reduce redundancy.
  • Easier to modify and manage.

5. What is data redundancy? How does DBMS help in reducing it?

Data Redundancy is the repetition of the same data in multiple places, which wastes storage and may cause inconsistency.

DBMS reduces redundancy as follows:

`
  • Centralizing data storage.
  • Normalizing tables to eliminate duplicate data.
  • Using relationships (keys) instead of repeated values.

6. Difference between Centralized and Distributed Database Systems

Centralized Database Distributed Database
In a centralized database system, all data is stored at a single central location. In a distributed database system, data is stored across multiple physical locations.
Centralized databases are easy to manage, but they are more vulnerable to system failures. Distributed databases are more reliable in case of failures, but they are more complex to manage and maintain.
Remote users experience slower access times in centralized databases due to the single access point. In distributed databases, local access is faster since data is located closer to users.
Centralized systems are generally less expensive to implement and maintain. Distributed systems require a higher setup cost due to the complexity of infrastructure and synchronization.

7. Who is Database Administrator (DBA)? What are the major responsibilities of DBA?

A Database Administrator (DBA) is a professional responsible for the installation, configuration, maintenance, and security of database systems. The DBA ensures that databases are operating efficiently, data is accurate and accessible, and users have the necessary permissions while maintaining the overall health and performance of the database infrastructure.

Major Responsibilities:

  • The DBA installs and configures the database management system (DBMS) to suit the organizationâ€™s requirements.
  • They manage user accounts and control access to the database to ensure that only authorized users can perform specific operations.
  • The DBA monitors the performance of the database to detect and resolve issues such as slow queries or system bottlenecks.
  • They are responsible for performing regular backups and implementing recovery procedures to prevent data loss in case of failure.
  • The DBA ensures the integrity and consistency of data by enforcing rules and constraints within the database system.
  • They apply updates, patches, and modifications to keep the database system up to date and secure from vulnerabilities.

Define normalization. Explain 1NF, 2NF and 3NF with suitable examples. Explain the normalization process with examples.

Normalization is the process of organizing data in a database to reduce data redundancy and improve data integrity. It involves dividing large, complex tables into smaller, simpler ones and defining relationships between them. The main goal of normalization is to eliminate duplicate data, ensure data dependencies make sense, and make the database more efficient, flexible, and scalable. This process follows a set of rules called normal forms, each with specific conditions to meet.

First Normal Form (1NF)

A relation is in 1NF if all the attributes contain only atomic (indivisible) values and each record is unique.

Example (Not in 1NF):

StudentID | Name | Subjects
--------------------------------
101       | Ram  | Math, Science
102       | Sita | English, Math

Here, the Subjects column contains multiple values, which violates 1NF.

Conversion to 1NF:

StudentID | Name | Subject
------------------------------
101       | Ram  | Math
101       | Ram  | Science
102       | Sita | English
102       | Sita | Math

Now, each attribute holds atomic values, and the table satisfies 1NF.

Second Normal Form (2NF)

A table is in 2NF if it is in 1NF and all non-prime attributes are fully functionally dependent on the entire primary key (i.e., no partial dependency).

Example (Not in 2NF):

StudentID | Subject | StudentName
-------------------------------
101       | Math    | Ram
101       | Science | Ram
102       | English | Sita

Here, StudentName depends only on StudentID, not on the combination of StudentID and Subject, so there is a partial dependency.

Conversion to 2NF:

Table 1: Student
StudentID | StudentName
------------------------
101       | Ram
102       | Sita

Table 2: Enrollment
StudentID | Subject
---------------------
101       | Math
101       | Science
102       | English

Now all non-key attributes depend on the full primary key, satisfying 2NF.

Third Normal Form (3NF)

A table is in 3NF if it is in 2NF and there are no transitive dependencies, i.e., non-key attributes should not depend on other non-key attributes.

Example (Not in 3NF):

StudentID | StudentName | Dept | HOD
--------------------------------------
101       | Ram         | CS   | Dr. Sharma
102       | Sita        | IT   | Prof. Joshi

Here, HOD depends on Dept, not directly on StudentID, which causes a transitive dependency.

Conversion to 3NF:

Table 1: Student
StudentID | StudentName | Dept
-------------------------------
101       | Ram         | CS
102       | Sita        | IT

Table 2: Department
Dept | HOD
-------------
CS   | Dr. Sharma
IT   | Prof. Joshi

This removes the transitive dependency, and the tables now satisfy 3NF.

9. Explain the terms: primary key, foreign key, and candidate key with examples.

Primary Key

A Primary Key is a field or a combination of fields in a database table that uniquely identifies each record in that table. It must contain unique values and cannot contain NULL values. The primary key ensures that each record is distinct and provides a way to reference specific records easily.

Example: Consider the following Student table where StudentID is the primary key:

StudentID (Primary Key) Name Age
101 Ram 18
102 Sita 19
103 Hari 17

Foreign Key

A Foreign Key is a field (or collection of fields) in one table that refers to the primary key in another table. It is used to establish and enforce a link between the data in two tables. Foreign keys help maintain referential integrity by ensuring that a value in one table corresponds to a valid record in the related table.

Example: Consider the following two tables, Student and Enrollment. The StudentID in the Enrollment table is a foreign key referring to the StudentID in the Student table.

Candidate Key

A Candidate Key is any field, or set of fields, that can uniquely identify a record in a table. A table can have multiple candidate keys, but one of them is selected as the primary key. Candidate keys must have unique values and cannot contain NULLs.

Example: In the following Users table, both Email and Username can uniquely identify a user and hence are candidate keys. One of them can be chosen as the primary key.

10. What is SQL? Explain its components and common functions.

SQL (Structured Query Language) is a programming language for storing and processing information in a relational database.

  • DDL (Data Definition Language): CREATE, ALTER, DROP
  • DML (Data Manipulation Language): INSERT, UPDATE, DELETE
  • DQL (Data Query Language): SELECT
  • DCL (Data Control Language): GRANT, REVOKE
  • TCL (Transaction Control Language): COMMIT, ROLLBACK

Common Functions:

  • Aggregate: COUNT(), SUM(), AVG(), MAX(), MIN()
  • String: UPPER(), LOWER(), CONCAT()
  • Date: NOW(), DATE()

12. Definitions of Key Database Terms

a. Data Dictionary

A Data Dictionary is a centralized repository of information about data such as meaning, relationships to other data, origin, usage, and format. It contains metadata (data about data) and helps in managing the structure and schema of a database.

b. Primary Key

A Primary Key is a field or a set of fields in a table that uniquely identifies each record in that table. It ensures that no two rows have the same value for the primary key and that the key is never NULL.

c. Relationship

A Relationship in a database refers to a connection between two or more tables based on common fields. Relationships help in organizing data efficiently and allow querying related data across multiple tables using keys.

d. Data Manipulation Language (DML)

Data Manipulation Language (DML) is a subset of SQL used to manipulate data within existing database objects. It includes operations like inserting, updating, deleting, and retrieving data. Common DML commands include INSERT, UPDATE, DELETE, and SELECT.

e. Structured Query Language (SQL)

Structured Query Language (SQL) is the standard language used to interact with relational databases. It allows users to define, manipulate, control, and query data using a consistent syntax. SQL includes DDL, DML, DCL, and TCL commands.

f. Data Integrity

Data Integrity refers to the accuracy, consistency, and reliability of data stored in a database. It ensures that the data entered into the database is valid, consistent, and free from errors through the use of constraints like primary keys, foreign keys, and unique constraints.

g. Data Definition Language (DDL)

Data Definition Language (DDL) is a subset of SQL used to define and manage database structure or schema. It includes commands such as CREATE, ALTER, DROP, and TRUNCATE, which help in creating and modifying tables and other database objects.

h. Data Security

Data Security involves protecting digital data from unauthorized access, corruption, or theft. In databases, this includes implementing user authentication, access controls, encryption, and backup policies to safeguard sensitive information.

i. Database System

A Database System consists of the database itself, the DBMS (Database Management System), and the applications or users that interact with it. It provides an organized way to store, retrieve, and manage data efficiently while ensuring integrity and security.

Define a void function fibonacci(int terms) that prints the series using iteration. In main(), call fibonacci(10);

void fibonacci(int terms) {
    int a = 0, b = 1, c;
    printf("Fibonacci series: ");
    for (int i = 1; i <= terms; i++) {
        printf("%d ", b);
        c = a + b;
        a = b;
        b = c;
    }
}

int main() {
    fibonacci(10);
    return 0;
}

A function in C is a self-contained block of code designed to perform a specific task. It serves as a fundamental building block for organizing and structuring C programs, promoting modularity, reusability, and readability.

Advantages:

  • Modularity: Divides a large program into smaller parts, making it easier to manage and understand.

  • Readability: Named functions clarify program logic and make the code easier to follow.
  • Easy Debugging: Errors are easier to locate and fix when the code is organized into functions.
  • Reduced Redundancy: Common tasks can be written once and reused wherever needed.
  • Maintenance: Updating code is easier since changes can be made in individual functions.
  • Abstraction: Users can use a function without needing to understand its internal details.

int power(int base, int exp) {

    if (num == 0) return 0;
